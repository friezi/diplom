\section{Peer-To-Peer-Systeme}
\label{Abschnitt:peer_to_peer}
Der klassische Ansatz, auf Daten oder Dienste innerhalb des Internets zuzugreifen, basiert auf dem Client/Server-Modell. Ein Server stellt als zentrale Einheit
Dienste zur Verfügung, die einzelne, voneinander unabhängige Klienten anfordern können. In zunehmendem Maße spielen Peer-To-Peer (kurz P2P) -Systeme eine wichtige
Rolle bei der Datenkommunikation im Internet. Ein $Peer$ ist eine Einheit, die sowohl Dienste zur Verfügung stellen als auch in Anspruch nehmen kann. So können
Peers beispielsweise Ressourcen teilen (Drucker, Speicherkapazität, ...) oder Daten direkt miteinander austauschen. Ein Vorteil von P2P gegenüber dem Client/Server-Modell
liegt in der Dezentralisierung von Ressourcen, so dass Ausfälle einzelner Einheiten  nicht unmittelbar den Zusammenbruch des Systems mit sich führen müssen.
Schollmeier unterscheidet in \cite{Schollmeier:P2PDefinition} $reine$ von $hybriden$ P2P-Systemen. Er definiert ein reines P2P-System als ein P2P-System,
bei dem eine beliebige terminale Einheit
aus dem Netzwerk entfernt werden kann, ohne dass das Netzwerk in seinen Diensten eingeschränkt wird. Bei hybriden P2P-Systemen sind zentrale Einheiten notwendig,
um Teile der angebotenen Netzwerkdienste zur Verfügung zu stellen bzw. verfügbar zu machen. Nichtsdestoweniger ist das Teilen von Netzwerkressourcen der Peers ein
substantieller Bestandteil eines hybriden P2P-Systems.\\

Verschiedene Netzwerkdienste arbeiten auf Basis von P2P-Systemen. Zu nennen wären beispielsweise Gnutella, Napster, Freenet oder KaZaA
(siehe \cite{Aberer:P2P,KaZaA}). Diese sind sogenannte {\itshape File-Sharing-Dienste} und dienen zum Austausch von allgemeinen Dateien oder
Musikdateien. Ein allgemeines P2P-System, was als Basistechnologie für andere P2P-Systeme dienen kann, ist Pastry \cite{Pastry}.
